#[no_mangle]
pub fn schedule(yield: bool) -> !
Expand description

Schedule the envs. If yield, the current env will be moved to the tail of the schedule list. Otherwise, the strategy will judge the rest time the env will enjoy.

We use the priority to represent the time-slice count of a env. If the count run out, the next env will be selected.
